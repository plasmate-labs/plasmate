Plasmate compiles HTML into a Semantic Object Model (SOM). It is built for throughput, token efficiency, and becoming a real open standard.

This is a standard, not a platform

Plasmate is open infrastructure. Apache 2.0. Built to be embedded everywhere: local tools, CI, sidecars, and future control planes.

The goal is not a flashy SaaS. The goal is that every serious agent stack ships a Plasmate-compatible path.

Install and run

Use the installer after the v0.1.0 release finishes. No Docker required. One binary, two modes.

Installcurl -fsSL https://plasmate.app/install.sh | sh
One-shotplasmate fetch https://plasmate.app
Serverplasmate serve --protocol awp
CDPplasmate serve --protocol cdp

Note: CDP is a legacy compatibility layer. AWP is the native agent protocol.

Why SOM beats DOM for agents

CDP was built for devtools, not for agents. The DOM is too large, too low-level, and too expensive in tokens.

SOM is the opposite: stable semantic regions, deterministic element IDs, content budgets, and interactive elements preserved even when the page is huge. Agents spend tokens on meaning, not markup.
