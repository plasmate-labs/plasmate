# Schema.org Proposal: WebPageSemanticRepresentation

**Status:** Draft Proposal
**Author:** David Hurley, Plasmate Labs
**Date:** March 25, 2026
**Related:** [W3C Web Content Browser for AI Agents Community Group](https://www.w3.org/community/web-content-browser-ai/)

## Abstract

This proposal introduces a new Schema.org type, `WebPageSemanticRepresentation`, to describe machine-readable semantic representations of web pages. As AI agents increasingly consume web content, a standardized way to indicate the availability and format of semantic page representations would improve discoverability, reduce redundant processing, and enable more efficient agent-web interaction.

## Motivation

AI agents currently interact with web pages by:

1. Fetching raw HTML
2. Parsing the DOM (often via headless Chrome)
3. Extracting meaningful content from presentational markup

This process is expensive (50,000+ DOM nodes per page), slow (2-5 seconds per page via Chrome), and wasteful (90%+ of tokens sent to LLMs are presentational noise).

Several projects now produce structured semantic representations of web pages:

- **Plasmate SOM** (Semantic Object Model) - JSON-based semantic structure
- **Jina Reader API** - Markdown-based page summaries
- **Firecrawl** - Structured page extraction
- **Accessibility trees** - ARIA-based semantic representations

A Schema.org type would allow websites to:
- Advertise that a semantic representation is available
- Specify the format and endpoint
- Indicate freshness and coverage

## Proposed Type

### WebPageSemanticRepresentation

A structured, machine-readable representation of a web page's content, optimized for consumption by AI agents and automated systems.

**Extends:** `CreativeWork`

### Properties

| Property | Expected Type | Description |
|----------|--------------|-------------|
| `representationFormat` | `Text` | The format of the semantic representation (e.g., "SOM/1.0", "markdown", "accessibility-tree") |
| `representationUrl` | `URL` | The URL where the semantic representation can be fetched |
| `sourceUrl` | `URL` | The URL of the original web page this representation describes |
| `tokenCount` | `Integer` | The approximate token count of the representation (using cl100k_base encoding) |
| `compressionRatio` | `Number` | The ratio of original HTML tokens to representation tokens |
| `generatedBy` | `SoftwareApplication` | The software that generated this representation |
| `contentScope` | `Text` | What content is included: "full-page", "main-content", "article-body", "navigation", etc. |
| `lastCrawled` | `DateTime` | When the source page was last fetched to generate this representation |
| `includesJavaScript` | `Boolean` | Whether the representation includes content generated by JavaScript execution |

### Example Usage

```json
{
  "@context": "https://schema.org",
  "@type": "WebPageSemanticRepresentation",
  "sourceUrl": "https://example.com/article/ai-agents",
  "representationUrl": "https://cache.plasmate.app/v1/som?url=https://example.com/article/ai-agents",
  "representationFormat": "SOM/1.0",
  "tokenCount": 1200,
  "compressionRatio": 14.2,
  "generatedBy": {
    "@type": "SoftwareApplication",
    "name": "Plasmate",
    "version": "0.4.0",
    "url": "https://plasmate.app"
  },
  "contentScope": "full-page",
  "lastCrawled": "2026-03-25T07:00:00Z",
  "includesJavaScript": true,
  "dateCreated": "2026-03-25T07:00:00Z"
}
```

### HTML Integration

Website owners could embed this in their pages to signal that a semantic representation is available:

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebPageSemanticRepresentation",
  "sourceUrl": "https://example.com/article/ai-agents",
  "representationUrl": "https://cache.plasmate.app/v1/som?url=https://example.com/article/ai-agents",
  "representationFormat": "SOM/1.0",
  "contentScope": "full-page"
}
</script>
```

### robots.txt Integration

Complementary to the Schema.org type, websites could indicate SOM availability in robots.txt:

```
# Semantic representations available
SOM-Endpoint: https://cache.plasmate.app/v1/som
SOM-Format: SOM/1.0
```

## Use Cases

### 1. AI Agent Discovery
An AI agent visiting a page could check for `WebPageSemanticRepresentation` markup before attempting to parse the full DOM, saving significant compute and token costs.

### 2. Search Engine Enhancement
Search engines could index semantic representations alongside traditional page content, improving the quality of AI-generated search summaries.

### 3. Semantic CDN
Content delivery networks could serve pre-computed semantic representations, reducing the need for each agent to independently parse the same pages.

### 4. Accessibility
Screen readers and assistive technologies could consume semantic representations directly, bypassing complex DOM traversal.

### 5. Content Archiving
Digital archives could store semantic representations as a compact, meaningful record of web content alongside traditional WARC files.

## Relationship to Existing Schema.org Types

- **WebPage** - `WebPageSemanticRepresentation` describes a representation OF a `WebPage`, linked via `sourceUrl`
- **CreativeWork** - The representation itself is a creative work (a structured interpretation of the source)
- **SoftwareApplication** - The tool that generated the representation, referenced via `generatedBy`
- **DataDownload** - Similar concept but `WebPageSemanticRepresentation` is specifically for web page content and includes agent-relevant metadata like token counts

## Implementation Path

1. **Community Group incubation** - Develop the type within the W3C Web Content Browser for AI Agents Community Group
2. **Schema.org proposal** - Submit as a pending type via the Schema.org GitHub repository
3. **Adoption** - Implement in Plasmate's SOM Cache, encourage adoption by other semantic web tools
4. **Iteration** - Refine based on real-world usage and feedback
